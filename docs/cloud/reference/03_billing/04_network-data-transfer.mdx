ClickHouse Cloud 会统计入站和出站的数据传输。
这包括所有进出 ClickHouse Cloud 的数据，以及同一区域内
和跨区域的数据传输。此类用量会在服务级别进行跟踪。
基于这些用量，客户会产生数据传输费用，并计入其
月度账单。

ClickHouse Cloud 会对以下内容收费：

* 从 ClickHouse Cloud 到公共互联网的数据出站，包括向相同或其他
  云服务提供商的其他区域进行的数据传输。
* 向同一云服务提供商中其他区域的数据出站。

同一区域内的数据传输，或通过 Private Link/Private Service Connect 进行的使用和数据传输不收费。
但是，如果我们发现某些使用模式影响了我们公平计费的能力，
则保留引入额外数据传输计费维度的权利。

注意：使用 Private Link/Private Service Connect 会产生云服务提供商（CSP）侧的费用。请参考相应 CSP 的定价页面以获取最新信息。

数据传输费用会因云服务提供商（CSP）和区域而异。
面向公共互联网的出站定价仅取决于源区域。
跨区域（或跨可用区）的出站定价则取决于源区域和目标区域。

**降低数据传输成本的最佳实践**

在处理入站和出站数据时，有一些模式可以帮助降低数据传输成本，需要特别注意。

1. 在 ClickHouse Cloud 进行数据入站或出站时，在可能的情况下使用压缩，
   以尽量减少传输数据量及相关成本。

2. 请注意，当通过原生协议、使用非内联值执行 `INSERT` 时（例如 `INSERT INTO [TABLE] FROM INFILE [FILE] FORMAT NATIVE`），ClickHouse
   客户端会从服务器拉取元数据以打包数据。如果元数据比
   `INSERT` 负载还大，那么从服务器的角度看，你可能会反直觉地看到
   出站流量大于入站流量。如果这不可接受，可以考虑
   使用 `VALUES` 语法将数据内联，或改用 HTTP 协议。
